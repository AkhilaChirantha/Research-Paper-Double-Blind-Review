Suggestions

  • Prioritize stronger experiments: add stronger baselines, ablations, dataset diversity, and clearer metric reporting.
  • Improve presentation before submission: make contribution bullets explicit, define notation early, and rewrite unclear method sections.
  • Sharpen novelty claims: say exactly what is new compared with the closest related work, not only that the method performs well.
  • Reduce technical-risk comments: state assumptions, proof scope, failure cases, and why the experimental evidence supports the claims.
  • Add reproducibility details: code link, hyperparameters, compute budget, seeds, implementation details, and data preprocessing.
  • Strengthen related work: compare directly against the closest papers reviewers are likely to cite.
  • Because this dataset has accepted papers only, add rejected papers later if you want a stronger true accept/reject classifier.
